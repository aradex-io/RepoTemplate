# Plan review

Non-trivial work is planned in `docs/plans/` and **reviewed before implementation**
(CLAUDE.md §4). The review is produced by a non-interactive, read-only reviewer and
written back into the plan file as an appendix, so it is version-controlled and
auditable next to the plan it judges.

This page is the low-level reference for `scripts/codex-review.sh` and the
reviewing agents.

## The contract

A review **must**:

- run **non-interactively** and **read-only** — the reviewer may read the repo but
  must not modify anything except to append the appendix;
- be produced via **`codex exec`** by default (the OpenAI Codex CLI), or a defined
  fallback;
- be **appended to the plan markdown** as a `## Appendix: Plan Review` section.

A review **must not**:

- use an **interactive** Codex session / TUI;
- use the **Codex "bridge"**, any **Codex MCP server**, or an IDE integration;
- be **fabricated** — if no reviewer is available, the tool stops.

## Usage

```bash
scripts/codex-review.sh docs/plans/2026-06-07-my-feature.md
```

It prints progress to **stderr** and appends the review to the plan file. Read the
appendix, resolve every **Blocker**/**Major** issue (revise the plan and re-run if
it changed materially), then implement.

## Backend resolution

`scripts/codex-review.sh` selects a backend, in this order, unless you force one:

| # | Backend | Selected when | Invocation |
|:-:|---------|---------------|------------|
| 1 | **`codex exec`** | `codex` is on `PATH` | `codex exec` (see flags below) |
| 2 | **fallback CLI** | no `codex`, but `$REVIEW_FALLBACK_CMD` is on `PATH` | prompt piped on **stdin**, review captured from **stdout** |
| 3 | **`plan-reviewer` agent** | no review CLI at all | the script exits `3` and instructs you to run the agent, which appends the appendix itself |

Force a backend with `REVIEW_BACKEND`:

```bash
REVIEW_BACKEND=codex   scripts/codex-review.sh docs/plans/<plan>.md
REVIEW_BACKEND=advisor REVIEW_FALLBACK_CMD=advisor scripts/codex-review.sh docs/plans/<plan>.md
REVIEW_BACKEND=none    scripts/codex-review.sh docs/plans/<plan>.md   # forces the exit-3 hand-off
```

## How `codex exec` is invoked

```bash
codex exec \
  --model "$CODEX_REVIEW_MODEL" \   # default: gpt-5.3-codex
  --sandbox read-only \             # Codex may read the repo, never write
  --skip-git-repo-check \           # run regardless of git state
  --output-last-message "$TMP" \    # capture ONLY the final message (not progress)
  "$PROMPT"                         # rubric + the full plan as context
```

- **`exec`** — one-shot, non-interactive mode. Progress streams to stderr; the
  final agent message goes to stdout (and to `--output-last-message`).
- **`--sandbox read-only`** — the enforcement behind "Codex reviews, it does not
  implement."
- **`--output-last-message`** — we append *only* this file, so stderr chatter never
  pollutes the appendix.

The Codex CLI authenticates via `codex login` or the `OPENAI_API_KEY` environment
variable; the script does not handle auth.

## The fallback CLI

A fallback reviewer is any command that **reads the prompt on stdin and writes the
review to stdout**. Point `REVIEW_FALLBACK_CMD` at it (default: `advisor`):

```bash
REVIEW_FALLBACK_CMD=advisor scripts/codex-review.sh docs/plans/<plan>.md
# internally:  printf '%s' "$PROMPT" | advisor > "$TMP"
```

This is the integration seam for any read-only LLM-review CLI you already use.

## Environment variables

| Var | Default | Effect |
|-----|---------|--------|
| `CODEX_REVIEW_MODEL` | `gpt-5.3-codex` | Model passed to `codex exec --model`. |
| `REVIEW_BACKEND` | `auto` | `auto` \| `codex` \| `advisor` \| `none` (or a custom command name). |
| `REVIEW_FALLBACK_CMD` | `advisor` | The fallback reviewer command (stdin→stdout). |

## Exit codes

| Code | Meaning |
|:----:|---------|
| `0` | Review produced and appended. |
| `1` | Plan file missing, or the reviewer returned an empty review (plan left unchanged). |
| `2` | Bad usage (no plan argument, or unknown `REVIEW_BACKEND`). |
| `3` | No review CLI available — run the `plan-reviewer` agent (plan left unchanged). |
| `127` | The selected backend's binary was not found. |

The plan file is only modified on exit `0`; failures never leave a partial appendix.

## The appendix

Appended verbatim to the end of the plan:

```markdown
---

## Appendix: Plan Review (2026-06-07T19:34:10Z)

<!-- Generated by codex exec (model: gpt-5.3-codex) — read-only, non-interactive. Do not hand-edit this appendix. -->

### Verdict
Approve / Approve with changes / Request changes — one sentence why.

### Strengths
### Issues            (numbered, Blocker > Major > Minor: problem, why, fix)
### Test coverage gaps
### Open questions
```

The HTML comment records which backend/model produced it. Commit the plan and its
appendix together (`docs(plan): ...`).

## The review rubric

The reviewer is prompted as a principal engineer doing a skeptical design review,
checking: correctness, completeness, hidden complexity, security, data/privacy
risk, failure modes, testability, rollout/rollback safety, and adherence to
CLAUDE.md. Output is constrained to the appendix structure above.

## The agents

| Agent | Role |
|-------|------|
| [`planner`](../../.claude/agents/planner.md) | Writes the plan into `docs/plans/`; does not implement. |
| [`codex-reviewer`](../../.claude/agents/codex-reviewer.md) | Runs `scripts/codex-review.sh`; on exit `3`, hands off to `plan-reviewer`. |
| [`plan-reviewer`](../../.claude/agents/plan-reviewer.md) | Fallback reviewer with no CLI: reviews the plan and appends the same appendix itself. |
| [`implementer`](../../.claude/agents/implementer.md) | Implements only after blockers are resolved. |

Slash commands `/plan` and `/codex-review` drive the first two steps. See
[agents & commands](./agents-and-commands.md).
